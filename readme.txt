分支和HEAD的概念我们以后再讲。

前面讲了我们把文件往Git版本库里添加的时候，是分两步执行的：

第一步是用git add把文件添加进去，实际上就是把文件修改添加到暂存区；

第二步是用git commit提交更改，实际上就是把暂存区的所有内容提交到当前分支。

因为我们创建Git版本库时，Git自动为我们创建了唯一一个master分支，所以，现在，git commit就是往master分支上提交更改。

你可以简单理解为，需要提交的文件修改通通放到暂存区，然后，一次性提交暂存区的所有修改。

俗话说，实践出真知。现在，我们再练习一遍，先对readme.txt做个修改，比如加上一行内容：

Git is a distributed version control system.
Git is free software distributed under the GPL.
Git has a mutable index called stage.
然后，在工作区新增一个LICENSE文本文件（内容随便写）。

先用git status查看一下状态：

$ git status

#   (use "git add <file>..." to include in what will be committed)

no changes added to commit (use "git add" and/or "git commit -a")

历史[编辑]
百度百科测试版上线于2006年4月20日，4月5日至20日为一段时间的内部测试，根据条目地址栏中的序号，最早创建的10个词条为百度百科、词条、编辑实验（沙盘页面）、馒头、兰花养植（因含错别字被删除）、雁荡山、灵峰、大龙湫、五大夫松和红色食品。2008年4月21日发布正式版。

截止到2016年1月百度百科的条目总量已超过1,313万，其条目增长基本经历了两个阶段，第一阶段是平缓增长阶段（2006年4月至2008年3月），第二阶段基本匹配指数增长形式（2008年3月后）[3]。

版面组成[编辑]
参见：维基百科:致其他百科用户#用词对照
首页[编辑]
百度百科当前的首页上半部分主要展示筛选出的知识性内容，下半部分主要展示优秀用户及其贡献，其中知识性内容多与当日或近期受到关注的新闻事件相关联，并列有相关的热点信息。从首页可以进入自然、文化、地理等11个分类频道页和数字博物馆、城市百科等专题页。

条目页[编辑]
百度百科的条目页分为普通页面和多义词页面两种形式，普通页面由名片、正文与辅助信息组成，多义词页面则分为主页面和义项页面，其中主页面只列举义项名称（其形式类似于维基百科的消歧义页面），义项页面则在普通页面的基础上增加了不同义项间跳转浏览的模块。条目使用的书写语言以中文简体为主，部分中文繁体、朝鲜汉字、日本汉字和越南汉字会被自动转换为中文简体[注 1]。

百科名片上线于2009年11月，包含条目概述和基本信息栏两部分[4]，前者类似文章的导语，对全文做概括性的介绍，字数限制为600字[5]，后者则以表单的形式归纳出一些基本信息和数据，两者均采用与正文分开的单独模块编辑。最初等级在4级以上、编辑通过率在85%以上的用户方能编辑百科名片，2013年9月这一权限全面开放[6]。

百度百科条目的正文长度要求控制在2万汉字字符以内（相当于4万字节），其中可设置两种级别的标题，系统自动为设置标题的条目生成目录。条目中最多可添加15张单张图片和20个图册（图册相当于图片集，每个图册可包含多张图片），另外也可以添加百度地图的动态截图和视频[注 2]。被直接引用的信息来源均列于页面底部的参考资料中，其中不区分脚注，此外由于图片页不能说明图片的来源情况，其中亦包含部分图片来源。

辅助信息集中在条目页的右侧和底部，大部分为系统自动设置，少量可以开放编辑（如部分联系表）。其中页面右侧的辅助信息包括词条统计、词条贡献榜、百科消息和推广链接等，其中词条统计包含页面浏览和编辑次数、最近更新时间和创建者等信息；词条贡献榜突出显示对条目进行编辑后获得优质版本或特色词条的用户，并采用不同的勋章图标标记；百科消息用于向用户展示一些最近的活动；推广链接则用于设置广告。页面底部的辅助信息包括图册、联系表和标签等。

图片页[编辑]
百度百科的图片页于2009年12月推出，页面左侧为图片（尺寸较大时默认按缩略形式显示，浏览时可选择放大至实际尺寸），其左下角打有百度百科的水印，下方为说明，但无图片来源、作者、版权状态和外部数据源等信息[7]。

专题页[编辑]
数字博物馆：为百度百科与中国大陆各大博物馆合作推出的线上博物馆展示平台，主要利用先进技术立体展示各博物馆藏品[8]。
城市百科：为大中华地区的城市专题页面（其中中国大陆地区以地级市为基本单位），2012年11月最早的城市百科成都站上线，2014年1月改版后按行政区划系统展示，主要以条目集合的形式列举每一城市的景点、饮食、文化和人物等。用户可于页面中提交相关条目，并经官方审查后收录。目前不少页面还处在建设阶段[9]。
条目组成与规范[编辑]
收录内容和分类方式[编辑]
与传统百科全书相比，网络百科全书所收录的内容一般更为宽泛，与中文维基百科、互动百科类似，百度百科亦将菜谱、影视作品、数码电子产品、网络程序和游戏等作为正式内容收录。但与维基百科所不同的是，百度百科并未严格界定百科全书与词典的不同，其中亦广泛收录字、词、成语解释，如“降”、“缠绵”、“一心一意”，诗词、文学作品，如“《池上》”、“《紫藤萝瀑布》”，并包含相当数量的商业产品（如“欧莱雅欢颜高水润醒肤乳”、“利福喷丁胶囊”，但现时因审核更严格，商业类条目的通过量已逐渐减少）和各类新闻事件（如“甜筒男”、“内裤男”）。百度百科当前已为商业类、人物类和医疗类条目创建了初步的关注度标准[10]，但在此之前已收录的相关条目数量庞大、难以复核，因此这一标准主要在新条目审核中执行。

除正式内容外，百度百科中也包含一些特殊条目，并作为正式内容计入到条目数量的统计中：

沙盘：即名为“编辑实验”的条目，用户可在此进行一些实验性质的编辑练习。
帮助：即百度百科中的常用术语，用于解释基本设置、规范和操作要领等，例如其用户头衔名称（实习小编、高级编辑等）。
用户页：被称为“百科权威认证ID”，被视为一种对百度百科或百度贴吧的高级用户进行“肯定和鼓励”的形式，只有极少部分用户有权创建[11][注 3]。
统计页：百度百科相关团队（明星团、高校组等）用于条目统计的页面[注 4]。
百度百科使用标签功能对条目进行分类，并支持按标签搜索，其分类方式主要针对条目的内容属性而非编写质量[12]（例如没有类似于维基百科的“小作品”分类）。百度百科已初步创建分类页和分类树，部分标签之间可设置层级关系，但相同含义的标签不能合并或重定向（如当前同时有“作者”和“作者简介”两个分类页）[注 5]。

条目命名[编辑]
与传统百科全书和其他网络百科全书类似，百度百科的条目名禁止出现错别字以及不规范的大小写（如DnA）、不规范的符号或标点等，其中不能包含修饰性的词语或附属说明（如伟大的中国、广东省广州市、毛泽东主席等，专有名词例外，如作为纪录片名称的“伟大的卫国战争”，但这一规定在实际执行中仍会出现大量反例[注 6]），亦不能使用非专有名词的句子和复合短语作为条目名。对于特定内容的命名则出台有针对性的规范，如诗词的命名要求包含格律或词牌名[13]。

多义词和同义词[编辑]

多义词内容格式
百度百科最初将“不同内容，同一命名（即一个条目名下包含多个概念意义）”的事物均写进同一个页面，2010年11月出台了多义词功能，对此类条目（称为多义词）逐步进行拆分，拆分后的条目每个义项（即条目的每一个不同的概念）为单独的模块，可以选择性的浏览和编辑[14]。但由于拆分后的义项不支持进一步分组，导致部分条目义项繁多、内容冗长、排列无序[注 7]。目前仅官方人员和部分级别较高的用户拥有拆分多义词的权限，普通用户则可以在贴吧中对需要拆分的条目和不合理的拆分进行反馈，拆分后普通用户均可创建添加新的义项，但仅官方人员可调整义项排序和删除义项。

百度百科的条目名称不可修改，“同一内容，不同命名”的情况则采用“同义词跳转”的方式解决，类似维基百科的重定向。当条目A被确定为条目B的同义词时，A被称为来源词，B被称为标准词，访问条目A时将被自动跳转到条目B[15]，条目A的内容则不能再访问和编辑。当前条目间、义项间以及条目与义项间均可设置同义词。同义词关系除官方人员设置外，亦可采纳百科蝌蚪团成员所提交的方案，其他普通用户则需要通过投诉反馈。另外，百度百科于2010年1月推出了搜索结果重定向跳转功能，会自动跳转到条目名所代表的大部分人所熟知的人物（或事物、事件）的标准名称，例如输入奥巴马跳转到贝拉克·奥巴马[16]。除标准词外的该条目的各个来源词（已创建的）也会作为正式内容计入已收录条目中。

百度百科一般以中文名称作为标准词，通常使用官方名称、存在特定命名规范的标准名称（如生物、药物学名）或公众认知度较高的通用名称（如深圳腾讯技术有限公司→腾讯，后者为标准词，下同）作为标准词，并一般采用全称（针对景点、组织机构等，但也有例外，如省市级地名以简称为标准词，省市级以下行政单位地名则采用全称），对于外文人名、地名的处理一般以内地较公认的习惯译名为主，亦少量采用台湾或香港的译名。部分条目则出台有针对性的规范，如事件通常使用时间+地点+事件名命名（5·12大地震→5·12汶川地震），并要求规范使用其中的空格和标点符号[17]。

模板和模板化[编辑]
百度百科中的模板指针对不同主题的条目内容所列出的标准结构框架，具有推荐性质，供用户编辑时参考，这里的模板实际上是一种讨论和经验积累形成的写作规范，但对于模板的使用没有强制要求[18]。自2011年11月起百度百科推出模板化策略，鼓励用户按照特定的目录格式编辑某一类别的条目[19]。除此之外，百度百科亦针对部分可以格式化整理的内容推出小型模板（称为模块），如电视剧类条目中的“分集剧情”[20]。

联系表[编辑]

唐朝百科关系内容
百度百科使用联系表创建条目与条目之间在定义性信息之外更深一层的关系，将单一条目组织串联为成体系的知识，类似于维基百科的模板。百度百科于2011年3月推出百科关系功能，可将某一条目作为中心词，以各种表格的形式广泛的归纳该条目与其他条目之间的关联信息[注 8]。联系表则部分以百科关系为基础生成，部分由管理员设置，主要显示于条目页底部，在2012年11月又推出了显示于页面右侧的联系表（称为主题模块）[21]，其中部分联系表为从中文维基百科中照搬而来[注 9]。

百科系统[编辑]
百度百科注册用户在阅读条目的同时也可以编辑条目，每一篇条目都有名为“历史版本”的页面记录着过去每一次的编辑。一些特别敏感或者是容易受到破坏的条目则会由管理员赋予不同程度的保护，借此避免用户作出被认为不恰当的编辑。

百度百科采用先审核后发布的模式，用户可以通过个人中心得知所提交编辑的审核情况，亦可以跟踪条目的最新修改（类似于维基百科的监视列表，限于用户本人创建或编辑过的条目，不可任意选定）。在条目内容受到破坏时，除少数用户拥有覆盖权（将某个历史版本覆盖为当前版本，借此撤销某一次或几次的修订[注 10]）外，大部分情况下需要用户投诉并由管理员处理，经常遭受破坏的条目也可以申请锁定保护。

搜索与编辑[编辑]
百度百科提供关键字搜索和标签搜索两种条目搜索方式，在搜索框中输入关键字后，点击“进入词条”可直接跳转至对应的条目页，如果出现搜索结果和欢迎创建的提示表明条目尚未收录，如果出现“抱歉，没有找到与‘XX’相关的百科结果”则表明该关键字已被屏蔽（类似于维基百科的白纸保护，不能创建）。对于多义词则一部分跳转至主页面，一部分跳转至某一被认为知名度较高的义项页面（如武艺、浮云）。点击“搜索词条”则可以搜索内容中包含这一关键字的条目。标签搜索的使用与关键字搜索类似，用于条目的分类搜索[22]。

无论是官方人员还是普通用户，在进行编辑操作时必须注册并登录。最初使用源代码编辑，在后续升级中转为所见即所得模式[23]，其编辑器设有字体、标题、内部链接、参考资料、表格、公式、地图等辅助工具，编辑时在页面右侧会给出编辑提示。在为条目配图时既可使用本地上传，也可使用百度图片搜索网络图片并进行网络上传[24]，编辑者可以选择图片排版方式，但图片大小为系统默认。在为条目添加和修改参考资料时需使用标注功能，参考资料的填写格式曾长期不做标准规定，至2012年5月方确立了较严格的规范。

百度百科中存在一部分以锁定的方式禁止用户编辑的条目，官方给出的解释为保证内容的公正、客观，避免可能存在的争议，一部分条目因容易遭受编辑战或恶意破坏而锁定（如高句丽、棒子），但除此之外大部分锁定的条目则涉及敏感政治（如民主、学潮），或与犯罪等影响社会稳定的因素相关（如枪支、炸药），另有一部分条目与百度公司自身相关（如百度、李彦宏），或经由与百度百科合作的各种专业机构编辑认证后作为权威内容锁定（如气象卫星、测地卫星），多义词则会因某一义项的锁定而导致其它义项连带锁定（如王刚）。另外因医疗类条目常常涉及被医疗机构利用进行广告营销的争议，百度百科在2012年12月推出“彩虹计划”，将4万个常见的此类条目锁定，只有认证过的专家拥有编辑权[25]。锁定阻碍了条目编辑的开放性和内容的及时更新，部分条目普通用户只能向管理员申请解锁后方能编辑，其他则只能提出建议，并等待管理员（或级别较高的用户）进行更新。

此外，百度百科自2009年4月起规定北京时间每晚23:00起至次日8:30止，除特殊紧急情况外，在此期间所提交的编辑停止更新，以避免影响用户正常的休息睡眠（后改为维护时间）[26]。如果用户在特殊时段编辑，则可以选择使用草稿箱等功能保存编辑成果，留待其他时段再行提交。

审核[编辑]
百度百科普通用户所修改提交后的版本均须经过百度内部的审核才能公开显示，审核按照百度百科规定的各类规则进行，时间长短不一，从几秒到几小时不等，一般使用系统过滤，[27]。未通过的版本可在用户个人中心的未通过区显示，并给出未能通过的理由。

百度百科中用户所提交的内容未能通过的原因一般包含两类，一类是被认为名称不匹配要求，内容不正确、不相关、不客观甚至出现违规（如广告、灌水或不良信息），或未达到百度百科所规定的质量门槛（例如图片辨识度不高或清晰度差），或删除了原有的被认为合理的内容，因此导致条目编辑后无变化或质量降低；另一类则是排版或编写格式不匹配规范要求，导致可读性差或不美观[28]。

管理和社区建设[编辑]

百度百科五周年用户聚会现场，2011年4月
为了便于用户投诉、建议和反馈，进行一定的社区建设，百度百科以贴吧的形式创建了一套管理系统，如百度百科吧、百科蝌蚪团吧等，百度百科投诉中心亦由贴吧的形式转变而来。此类贴吧多由官方人员和一些较活跃的资深用户共同管理，用户可以在此提出问题、意见、反馈、建议，或寻求编辑帮助（如因等级不够而无法编辑条目），或相互之间进行讨论和交流。

目前最常用的管理平台为2009年8月启用的百度百科投诉中心，可受理用户的各类投诉、意见和建议，包含举报不良信息、未通过词条申诉、封禁查询与解封和反馈侵权信息四类[29]，投诉采用用户与管理人员一对一的方式进行，用户不能搜索、查看或参与他人的投诉，在解决后会通过站内消息告知用户处理结果，并允许用户对此申请复议。除此之外目前较为活跃的贴吧有百度百科吧（主要用于收集意见和建议，也可受理投诉）、百科蝌蚪团吧、百科优质版本吧（仅用于优质版本的推荐、评审和受理相关投诉）和百科编修院吧等。

除贴吧外，百度百科官方与用户以及用户间的线上讨论和交流也常常使用百度的另一个产品百度Hi进行，用户亦可以通过参加百度百科所主办的各类活动获得线下聚会的机会，其中规模较大的为周年庆典时所举办的线下聚会，受到官方邀请的用户可以参加庆典并与相关员工进行面对面的交流，此活动自2010年起连续举办。

质量提升措施[编辑]
百度百科早期质量提升的主要形式为百科任务（类似于中文维基百科的动员令），为管理员或百科蝌蚪团成员发起，所有用户均可参与的主题活动，每期持续约1至3周不等。期间用户可以编辑任务集合已有的条目，也可以围绕主题创建添加新的条目。部分任务通过与商家或社会机构、组织合作的形式举办[注 11]。为了鼓励用户参与，每期任务中贡献最大的几位参与者会获得实物奖品[30]。2010年后百科任务一度长期停办，2013年11月改版后重新推出，改由四级及以上用户发起，并由发起者规定任务目标和参与者的范围，百度百科督察院成员可参与审核，官方则根据任务的难易程度给予发起者和参与者奖励[31]。

除百科任务外，主题任务为另一质量提升的主要形式，此类任务要求参与者完善条目中某一特定的组成部分[注 12]，用户在领取并完成系统所分配的任务包后可获得活动积分，继而兑换虚拟徽章或实物奖品。但由于任务参与者的编辑水平参差不齐，且部分用户以积分、奖品为首要目的，因此也不可避免的导致众多条目质量不升反降。

为提高条目内容的专业和可信程度，百度百科以合作的方式邀请一些知名品牌、专业机构对少量条目进行编辑和认证（称为权威合作）[32]，或提供图片、视频或数据等形式的网络资源（称为资源合作）[33]。此外百度百科创建了名为百科学术委员会的组织[34]，经过认证的专业人士可以对极少量条目的内容进行判断，并对存在争议的部分提供专业意见，但存在一定程度的尸位素餐和跨领域判断的现象[注 13]。

模范树立措施[编辑]
优质版本为百度百科长期以来树立条目质量模范的主要方式，包括针对条目内容的优质（分初中高三个级别）和针对条目图片的优质，评选主要基于条目编辑前后变化量的对比，并从内容、格式、知识性、规范性、可读性等几个方面考量，对某些问题则出台细节性的规则（如时效性措辞）。除初级优质（原高质量版本）可以通过编审系统少量预选外，一般需要用户通过自荐和参与人工评审获得，并通常需在官方评审人员的建议下反复修改和申请，获得此类版本的用户被称为“突出贡献者”[35]，其用户名可在条目浏览页右侧永久性荣誉展示，除非条目被删除。但优质版本只是代表条目前后质量大幅提高，并不代表其内容正确、全面。一些编辑者利用规则，选择容易申请的某类条目（如诗词、图书、村庄）批量化的创建和申请，使得此类版本并不能均等地反映和衡量用户的工作量，另外由于规则变化时并不进行复审，不同时期的优质版本亦质量不一。

2013年4月，百度百科在优质版本的基础上推出特色词条（最初称为优秀词条），针对条目的整体质量作出评选并对申请用户进行荣誉展示[36]。7月又推出特色词条维护者，对某一特色词条贡献足够多、具有良好的维护能力的用户可获得该条目单独的免审权和覆盖权[37]。优质版本及优秀词条不一定是优质，因为这些词条要由用户自己推荐，意味着一些优秀的词条，因为没有推荐而不被定为优质版本。

用户状况[编辑]
百度百科的大部分内容由百度用户编写，一部分用户以完善、梳理、订正条目信息和学习、分享知识为目的，为百度百科的主要贡献者。等级达到四级，通过率达到85%，复杂编辑达到50个的用户被称为核心用户，可获得特有的形象和徽章等[41]。一些长期参与、表现突出和获得官方认可的用户可以加入不同的用户团队，并获得更多的编辑权限。另一部分用户的编辑则主要是为了其他目的，如赚取积分和奖品、广告宣传或破坏恶搞等。

除用户外，一些官方管理人员也参与百度百科编辑，其职责范围包括更新被锁定的条目、拆分多义词、添加联系表和视频等。官方人员也可以运行网络机器人进行一些简单的创建和编辑操作，如复制特定数据库的内容创建条目、添加内部链接、地图和模块等，常用的机器人为百科ROBOT[注 15]和w_ou，官方以外的普通用户则没有运行机器人的权限。

2013年的一项研究表明，百度百科用户平均创建条目数量为2.2，其中2.37%的用户创建了占总量34.9%的条目，平均编辑次数则为1.89，其中4.82%的用户贡献了43.93%的内容，即少量的用户集中贡献了大量的内容，大量分散的用户贡献率较低，此外用户对于创建和编辑两种行为基本没有偏好性[3]。另一项研究表明，百度百科用户的参与意向主要受到其有用性感知（百度百科的工具书属性对于用户工作、学习的帮助）、态度（用户喜欢使用百度百科的程度）和兴趣动机的直接显著影响[42]。

用户团队[编辑]
百科蝌蚪团[编辑]
百科蝌蚪团计划推出于2007年4月，为百度百科最早的用户团队计划，旨在招募部分愿意参与百度百科日常维护的优秀用户来协助管理百度百科，这些用户被称为“蝌蚪”[43]。与普通用户相比，“蝌蚪”拥有直接提交同义词的权限，并可进一步申请拆分多义词等更多的权限[44]，以及与管理人员更直接的沟通渠道，并可通过持续的贡献获得专属的荣誉和奖励。但同时“蝌蚪”需每隔一段时间参加考核，连续半年无贡献的成员会被清退[45]。

由于“蝌蚪”多为百度百科中较优秀和活跃的用户，百科蝌蚪团在百度百科起到了一定的表率作用，这一团队所主导的讨论和交流亦促进了百度百科系统与规则的完善，但“蝌蚪”的能力依然参差不齐，相当一部分“蝌蚪”对于版权并不比普通用户更在意。也有一些用户认为这一工作是负担，对百度百科的热情和坚持减淡，以至自行离团。

存在问题[编辑]
争议[编辑]
主条目：百度相关争议
开放性争议[编辑]
百度百科声称自己为内容开放、人人可编辑的网络百科全书，并提倡用户间平等、协作的精神，一般中国大陆的网站在比较百度百科及维基百科时，也都会强调“百度百科”与“维基百科”一样是一个“让所有网民参与编撰的网络百科”[49]。但百度百科对用户的编辑权限进行了部分限制，一些权限需要用户在达到一定的贡献量和活跃程度后单独申请，少量名称被长期白纸保护、内容被长期锁定的条目则无法被官方以外的用户所创建或编辑[50]。百度百科的管理人员由百度雇佣和任命的方式产生，与其相比普通用户所能争取到的管理权限也较为有限，仍处在辅助的地位。与百度百科的发展所相关的政策决定并非通过公开的讨论、评议和投票的方式产生，普通用户对此缺乏影响力和话语权。

因版权问题，百度百科的内容并不匹配各种公认的有关开放或自由的定义[51][52][53][54]。其系统所使用的共笔写作软件并非开放源代码软件，条目内容所使用的文件格式也不是开放格式，导致用户无法访问内容的源代码，也无法自由导出条目数据（包括使用API输出）。

版权争议[编辑]
按照百度百科的权利说明，百度百科规定用户所发表的内容版权归原作者所有，若为转载其版权归属则以原作者或原网站版权声明为准，一旦发表后，用户同意百度百科对其享有免费的、永久性的、不可撤销的、非独家的和完全再许可的权利和许可，百度百科进而在条目最底处均标示了版权所有的声明：“©2016 Baidu”。但百度百科用户从受版权保护的网站或正式出版物上转载内容时普遍没有列明来源和标示原作者或原网站的版权声明，或完全忽视其授权条款，造成对他人著作权的侵犯[注 16]。百度虽然也开通了版权投诉，但由于需要书面投诉，故实际投诉百度的人并不多。这导致其他受到版权保障的各类文章及多媒体内容，或是原属网络上公有领域、限制部分版权（例如：Creative Commons或是GFDL等授权）模式下的自由版权内容，成为百度的私有财产，或蒙受被解读为百度私有财产的危险。

此外，百度百科的版权声明中含有下述内容：

3.百度百科的用户在百度百科上发布的内容引用自CC协议（“知识共享”协议，经本地化的中国大陆版许可协议的中文名称为“知识共享2.5中国大陆版协议”。署名－非商业性使用－禁止演绎。参见http://cn.creativecommons.org/ 。共包括六套不同的协议，供使用者选择，以满足不同的需要。……关于六套协议的具体信息，请参阅http://cn.creativecommons.org/index.php/licenses/licenses_exp 。）或GFDL（GNU自由文档协议，GFDL下内容及其衍生品，可以自由地下载、复制、展示、再分发及商业使用，用户引用时请明确给出出处的链接，且只能继续以GFDL授权释出。参见http://www.gnu.org/copyleft/fdl.html ）保护的作品的，应分别按照上述协议的规定，合理使用他人作品。

很多人因此以为百度百科上发布的内容引用自CC协议或GFDL，但是去括号以及里面的内容后，条文变成“3.百度百科的用户在百度百科上发布的内容引用自CC协议或GFDL保护的作品的，应分别按照上述协议的规定，合理使用他人作品”，并无说明百度百科的版权状态。

中立争议[编辑]
百度百科并没有严格要求需要中立，因此许多条目存在不中立的问题，如一些景点的广告宣传内容等。有些不中立的条目还禁止更改。如：中华民国。并且因为百度百科大多是中国大陆人编写的，所以许多条目往往存在中国中心主义，经常出现“我国”“国产”等词语。以及中国民族主义、大汉族主义，例如库页岛、新疆、西藏、台湾等条目会有以粗体字标明“某某自古以来是中国领土”或“自某朝以来就是中国不可分区的一部分”之类的非中立性用词，而并没有客观的从各国或各民族观点书写。

编审相关[编辑]
审核标准和质量[编辑]
百度百科对涉及政治敏感的条目内容审核较为严格[55]，以避免“恶意评价国家现行制度，攻击政府”的内容[56]，部分条目因此类政治审查而违反中立原则，如条目2010年诺贝尔奖不允许写明2010年诺贝尔和平奖得主为对中华人民共和国当局持不同政治意见的刘晓波[注 17]，并在这一消息公布后删除了大量编辑版本，而其他类别的条目审核则较为宽松，并且包容一些侵权行为[注 18]。由于使用机器参与审核，而机器无法准确识别材料的可靠性、准确性和重要性，加上开发人员自身知识体系的不尽完善，往往使存在主观性、个人原创研究和误导的内容轻易通过，这一问题在互动百科、搜狗百科等类似网站中亦普遍存在。 由于人物评审下放用户，出现了评审标准不一，某些与拥有评审权用户良好关系的用户编辑的词条（优秀词条等）更容易获得通过，由于审核属于大多是机器过滤，出现了2014年五一劳动节期间由于系统维护出现大量版本积压没有审核，五一劳动节过后才陆续恢复正常。

同义词[编辑]
由于用户在百度百科创建条目过于容易，创建后又主要靠用户投诉、管理员判断确认后合并同义词[注 19]，有时还要用户自己提供相关证明材料，所以条目存在大量的重复，特别是外国人名、地名、影视作品等。例如2008年的动画片电影机器人总动员在百度百科就有Wall·E、Wall.e、WELL/E、WALL E、Wall-E、机器总动员、机器人总动员7个条目之多[注 20]，另有众多如“北京市第四中学”与“北京四中”这样全称与简称条目并存的现象。百度百科虽然鼓励用户对同义词的内容进行合并，但仍有大量条目并没有做到完全集成。

刷分[编辑]
百度百科中部分用户为了迅速赚取积分、提高等级、进阶核心用户或在活动中获取奖品[注 21]，为获得5分的最高奖赏，常常大量删除百度百科的内容、内链、放与词条无关的图片、复制粘贴等无意义编辑，令词条质量下降。常常利用其在版权审查方面的不作为和关注度制定方面的不完善，大量复制粘贴其他网站的内容批量创建条目，其复制粘贴的对象包括售卖网站的图书与电子数码产品介绍、汉典中的词语解释、鉴赏辞典中的诗词歌赋鉴赏和其他网络百科全书的新条目等等。添加无关或关联较弱的内容（如在景点类条目中详细介绍城市）和以错误的内容凑数等等，受其影响，一些频繁的进行细小编辑的用户也有可能被指责为“刷分”，一些微小的编辑也可能以“词条编辑前后无变化或编辑后质量降低”为由被拒绝通过。

此外，复制粘贴也导致百度百科中存在大量没有经过一定数量的用户协作写作、不够成熟的条目，2013年的一次统计分析表明，百度百科中条目的平均编辑次数为3.5次，其中创建后就再没有被修订过的条目占总量的50.06%，编辑次数小于4的条目占总量的78.74%[3]。

广告宣传[编辑]
虽然百度百科的条目编辑原则要求客观叙述、禁止出现广告[注 22][57][58]，但由于百度百科的条目在百度搜索中的优势地位，同时百度百科的关注度标准不够完善，加上审核的局限，这都在一定程度上造成了一些网站利用百度百科做推广[59]。除在条目内容或参考资料中添加广告链接外[注 23]，也常常直接以公司、企业、产品为名直接创建条目[注 24]，同时网络上亦出现了商业化的核心用户ID售卖和条目写手[注 25]。个别的广告宣传活动甚至以商家与百度百科合作的方式进行，与香港、台湾等地的杂志不同，活动中所撰写的广告宣传文章（软文或鳝稿）完全为正式的条目，没有任何标示告知读者为宣传文章[60][61][62]。

2012年12月，多家媒体披露了百度百科上部分医学类条目（比如子宫颈糜烂）存在不实信息，有误导用户之嫌，而个人用户前去修改时审核未能通过，但某私立医院所属的编辑者所做的修改却予以通过，引起了多方对百度百科是否具有公正性的讨论[63]，北京协和医院副教授龚晓明称百度百科“成了私人医院的营销战场”，另据媒体报道，杭州一位患者因轻信百度百科耽误治疗而去世[64]。对此百度百科出台“彩虹计划”，将大约4万个常见的医学条目锁定，只允许获得百度专业医师认定的注册用户编辑，而对于那些没有经过百度专业医师认定的编者所编写的条目，则提示“本词条涉及医疗卫生专业知识，认证工作正在进行中，当前内容仅供参考。”[65][66]

破坏恶搞[编辑]
更多资料：神兽 (恶搞)

2013年8月成都条目被恶搞的截图（红框处），添加了“中华人民共和国首都”、“中国第一大城市”等失实信息，国内生产总值等数据也被篡改。
百度百科长期受到破坏和恶搞问题的影响，通常情况下出现在有争议或大众性的条目中（如公众人物），用户或删除、清空不匹配其价值观的文字，或插入虚假信息，或进行调侃以至诋毁[67]。也有一些地方主义者或持有排外观点的用户对城市和行政区划类条目进行破坏恶搞，或鼓吹各自地方，篡改客观真实内容，或恶意丑化其他地方[注 26]。

由于百度百科中的条目内容很多是经复制粘贴而来，故频繁被网友上传网络流行的恶搞文章，如恶搞革命烈士的条目“史习焜”[68]，城管曾被解释为“形容残酷、血腥、恐怖”、“等同于打、砸、抢”[69]，专家曾被解释为“吃饱了没事干或正事一件都没干过”[70]。由于这些恶搞已是网络上流传已久的段子，所以大多数编辑者对这种错误的解释视而不见，表示默认[71]，多数条目在通过一段时间后才被投诉处理。

一些出现在百度百科中的恶搞行为进而对其他媒体产生了误导，如条目“布拉格之春”[注 27]曾被用户将《红色警戒2：尤里的复仇》中“尤里的复仇”一词恶搞为正式的军事活动代号，并在上海电视台纪实频道《中苏外交档案解密》节目中被当做史实引述[72]，条目“航母杀手”[注 28]中用户将英伟达Geforce系列显卡GTX690恶搞为“690战术核显卡”，同样被甘肃卫视《解密真相》节目原文引用，引起一片哗然[73]。

2009年1月初，一篇名为“百度十大神兽”的文章在网上发布并形成网络快速传播现象，网友将一些涉性内容、不雅用语按照谐音恶搞为事实上不存在的动物，并在百度百科中发布，如草泥马、达菲鸡、法克鱿、尾申鲸等[74]，被删除后又相继出现“十大神秘美食”、“十大神器”等后续作品[75]，使百度百科受到广泛质疑，但现状条目已全部被删除并白纸保护。

部分用户也常常针对系统过滤式审核的缺陷，创作一些内容看似学术严谨但实为伪科学的条目，或以明显违反科学常识、夸夸其谈的语调来对专业条目进行讽刺恶搞，例如化学条目超盐酸、聚甲烷和卡元素[76]。2014年4月“对二甲苯”条目更是由于“低毒”属性被篡改为“剧毒”而爆发编辑战，引发媒体广泛关注[77][78][79]。

随着审核质量的改进和投诉的普及，此类破坏行为的发生频率有所降低，但仍会有一些破坏长期不能被发现。此外，由于百度百科不设置用户检查功能，普通用户并不能查询破坏者的IP地址，因此对于用户注册多个分身进行破坏的现象难以要求管理员及时作出有效封禁。

技术相关[编辑]
百度百科系统因自动繁简转换问题无法显示繁体字，无法识别所有编码，一些非英语语言如西班牙文、葡萄牙文中的特殊字母和符号如“ñ, ã, ¡, ¿”等无法正确显示，而是显示为NCR代码[注 29]。一些日文汉字、生僻字也会因自动繁简转换问题不能正确显示，而被转换为汉语简体字（如输入幵会被转换为开）。

在2013年4月发布数学公式编辑器之前，百度百科系统长期缺少公式编辑工具，一些理工学科的公式不能准确输入并合理排版显示，导致一些编辑者习惯直接以截图的形式发布[注 30]，目前这一状况已得到改善[80]。

评价[编辑]
正面观点[编辑]
新华网观点认为“百度百科在网络中的影响力日益深远，并逐渐凸显出其作为互联网平台对公众知识的传播价值”[81]。随着其内容的充实和完善，百度百科也逐渐被大陆媒体作为资料引述[注 31]，并获得一定的认可，例如2014年一项对南京地区大学生群体使用Wiki情况的调查表明，百度百科为调查群体中最为熟悉的Wiki产品，且对百度百科的使用情况总体评价为满意和非常满意的占六成以上（61.4%，中文维基百科为22.3%）[82]。

负面观点[编辑]
在版权争议之外，对百度百科的批评主要集中在学术性和中立性等方面，前者表现在其收录的条目普遍缺乏详细清晰的来源参考，与列明来源这一学术准则相悖，也使得条目的准确性和可信性受到质疑[83]。一些条目的内容自其他网站上直接拷贝而丝毫不加考证，亦包含一定的个人原创研究。后者表现在基于政府对互联网进行网络内容审查的背景，一些涉及中国历史、政治的条目或有明显的政治倾向，或因涉及政治敏感而禁止编写，如“六四事件”被禁止创建[注 32]。

2011年一项有561名百度百科用户参与的网上投票表明，用户自身对于百度百科的主要批评包括“审核不足，失误内容泛滥”、“太多抄袭粘贴，太少原创”和“词条信息缺乏权威可信性”等[注 33]。

与类似项目的关系[编辑]
与其他网络百科全书的比较[编辑]
在与维基百科的比较方面，一些相关的研究观点包括：

定位：对维基百科而言，Wiki的理念和模式体现在整个社区的价值观，旨在为社区创造更多的价值，而对百度百科而言，作为搜索引擎的附属产品，Wiki的理念和模式服务于百度公司的搜索引擎发展战略，即构建搜索社区化时代。[84]
管理方式：维基百科完全为自组织，所以更多的需要创建明确的、公开的站务管理规则（自我管理规则），而百度百科则是自组织与他组织的结合，但管理员和内容审核员均由百度公司员工担任，用户权限非常有限，所以缺乏对于规则的建设。[84]
运营方式：维基百科的收入主要来自于捐赠，百度的社区和搜索紧密相连，百度百科丰富了百度社区的内容，并通过积累用户个性化数据使得精确定位广告成为可能。[84]
激励机制：对于维基百科用户而言，参与公共事务、分享知识起到了主要的激励作用，百度百科则采用精神激励与物质激励相结合，虽然导致了刷分现象，但适应中国大陆国情，使得条目数量可以实现爆炸式增长。[84]
条目增长：百度百科创立时间比互动百科晚10个月，比维基百科晚3年半，在百度百科创立时中文维基百科已存在且有65,000个条目[85]。但在初创时期百度百科的条目数量平均增长率明显高于维基百科，按照产品生命周期理论解释，百度百科跳过了Wiki产品的导入期，直接进入高速成长期。[84]
条目内容：从内容分布而言，百度百科侧重于与人们日常生活紧密相关的条目，中文维基百科则侧重于自然科学。从叙述方式而言，百度百科更为详细，围绕条目主题尽可能将其所有相关信息罗列，中文维基百科则多涉及其基本信息，其他信息则通过参考文献、参见、外部链接等方式体现。[86]除此之外，百度百科可以有某百度贴吧的词条，如李毅吧、化学吧，但维基百科会将百度贴吧的词条视为广告，因此维基百科不能有具体某百度贴吧的条目。
编辑功能：百度百科试图创造直观、简便、易操作和宽松的用户环境，以简单取胜，降低用户入门难度，但丧失了一部分专业、有责任心的用户。[84]
审核机制：维基百科更侧重版权保护与原创性，百度百科则更侧重政治敏感性，导致内容的中立性和质量的全面性有所欠缺。[84]
